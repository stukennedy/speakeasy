## Build & Development Commands

```bash
# Development
irgo dev                    # Dev server with hot reload (preferred)
make dev                    # Same, falls back to ./dev.sh if irgo not installed
go run . serve              # Quick server without file watching

# Build
templ generate              # Regenerate *_templ.go from .templ files (MUST run after editing .templ)
make build                  # templ generate + go build
make templ                  # Just regenerate templ files

# Test
go test -v ./...            # Run all tests
go test -v -run TestName ./handlers/  # Run a single test

# Desktop
irgo run desktop --dev      # Desktop app with devtools

# Install toolchain
make install-tools          # Installs templ, air, gomobile, irgo CLI
```

## Architecture

This is an **Irgo** app — a hypermedia-driven framework using Go + Datastar + Templ. The server returns **HTML fragments via SSE**, never JSON. Datastar on the client handles DOM patching.

```
Browser event → Datastar HTTP request → Go handler → Templ renders HTML → SSE response → Datastar patches DOM
```

### Key Packages (from `github.com/stukennedy/irgo`)

- `pkg/router` — HTTP router with chi/v5 underneath. Two handler signatures:
  - `r.GET/POST(path, func(ctx) (string, error))` — full page loads, return HTML string
  - `r.DSGet/DSPost/DSPut/DSPatch/DSDelete(path, func(ctx) error)` — Datastar SSE endpoints, use `ctx.SSE()` for response
- `pkg/render` — `NewTemplRenderer()` renders templ components to HTML strings
- `pkg/testing` — `irgotest.NewClient(handler)` for HTTP testing; `.Datastar()` for SSE request testing

### Project Layout

- `app/app.go` — Router setup, static file serving, route registration
- `handlers/` — HTTP/SSE handlers; `Mount(r)` registers Datastar routes
- `templates/*.templ` — Templ source files (type-safe HTML components)
- `templates/*_templ.go` — Generated Go code (do not edit; regenerated by `templ generate`)
- `static/` — Embedded CSS/JS via `//go:embed`; Tailwind CSS generated from `input.css` → `output.css`
- `main.go` — Web/mobile entry (`//go:build !desktop`)
- `main_desktop.go` — Desktop entry (`//go:build desktop`)
- `mobile/mobile.go` — gomobile bindings for iOS/Android

### Handler Patterns

**Standard handler** (full page): returns `(string, error)` with rendered HTML.

**Datastar handler** (SSE): returns `error`, uses `ctx.SSE()` methods:
- `sse.PatchTempl(component)` — patch DOM with rendered templ component
- `sse.PatchHTML(html)` — patch DOM with raw HTML string
- `sse.PatchSignals(map)` — update client-side signals
- `sse.Remove("#id")` — remove element by CSS selector
- `sse.Redirect("/path")` — client-side navigation

**Context input**: `ctx.Param("id")`, `ctx.Query("q")`, `ctx.ReadSignals(&struct)`, `ctx.FormValue("name")`

### Testing Pattern

```go
client := irgotest.NewClient(app.NewRouter().Handler())
resp := client.Get("/")               // Standard HTTP
resp := client.Datastar().Get("/api")  // Datastar SSE request
resp.AssertOK(t) / resp.AssertSSE(t) / resp.AssertContains(t, "text")
```

### Templ Workflow

1. Edit `.templ` files in `templates/`
2. Run `templ generate` (or `irgo templ` / `make templ`) to regenerate `*_templ.go`
3. Never edit `*_templ.go` files directly
4. `irgo dev` auto-regenerates on save

### Datastar Conventions

- Elements that get patched via SSE **must have `id` attributes**
- Client state lives in `data-signals="{key: value}"`
- Two-way binding: `data-bind:signalName`
- Server requests: `data-on:click="@get('/path')"` (also `@post`, `@put`, `@patch`, `@delete`)
- Event modifiers: `__prevent`, `__debounce.300ms`, `__throttle.100ms`, `__once`
- Conditional display: `data-show="$signal"`, `data-class:name="$expr"`

### Build Tags

- Default build (`go build .`) → web/mobile using `main.go`
- Desktop build (`go build -tags desktop .`) → uses `main_desktop.go` with webview
- `irgo run desktop` adds the tag automatically
